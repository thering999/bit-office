CLAUDE.md — สมองของ Claude Code
ไฟล์ CLAUDE.md เป ็ น"หน่วยควำมจ ำถำวร" ของ Claude Code
ที่จะถูกโหลดทุกครั้งที่เริ่ม session ใหม่ ใช้บอก context,
coding standards, และ project rules ให้ Claude รู ้
ต าแหน่งไฟล์:
~/.claude/CLAUDE.md Global — ใช้กับทุกโปรเจกต์
./CLAUDE.md Project root — context เฉพำะโปรเจกต์
./frontend/CLAUDE.md Subfolder — context เฉพำะ module
ตัวอย่าง CLAUDE.md
# Project: MyApp
## Build Commands
- npm run dev (development)
- npm run test (run tests)
- npm run lint (check code)
## Code Style
- Use TypeScript strict mode
- Functional components + hooks
- Use Zustand for state mgmt
## Architecture
- src/components/ → UI
- src/stores/ → State
- src/api/ → API calls

เทคนิคการ Prompt ที่ได้ผลจริง
กฎทอง: ใช้ Plan Mode ก่อนเสมอ!
กด Shift+Tab เพื่อเข้ำ Plan Mode → ให้ Claude วิเครำะห์ป ั ญหำส ำรวจทำงเลือก และวำงแผนก่อน แล้วค่อยให้เขียนโค้ด ยิ่งวำงแผนดี
ผลลัพธ์ยิ่งแม่นย ำ
1
 วางแผน "วิเครำะห์ codebase แล้ววำงแผนกำรพัฒนำ feature X ให้หน่อย อย่ำเพิ่งเขียนโค้ด"
2
 Review แผน ตรวจสอบแผน ถำมค ำถำมแก้ไข assumptions จนพอใจ
3
 สั่งท า "เริ่มเขียนโค้ดตำมแผนได้เลย" — ใช้ "think hard" ส ำหรับงำนซับซ้อน
4
 ตรวจ+ทดสอบ Review output, รัน test, ถ้ำมี error ก็ paste error message ให้ Claude แก้

Spec-Kit — Spec-Driven Development
Spec-Kit by GitHub — เครื่องมือส ำหรับ Spec-Driven Development ที่ก ำหนด "อะไรจะสร ้ ำง" ก่อน แล้วค่อยสร ้ ำงลดกำรเขียนโค้ด
มั่วๆ ให้ AI ท ำตำมspec อย่ำงเป ็ นระบบ
1
 /speckit.constitution ก ำหนดหลักกำรโปรเจกต์(9 principles): Library-First, Test-First, Simplicity...
2
 /speckit.specify เขียน spec ว่ำต้องกำรอะไร ไม่ต้องคิดเรื่อง tech — pure product specification
3
 /speckit.clarify แก้จุดที่ยังไม่ชัด [NEEDS CLARIFICATION] ตอบค ำถำมท ำproduct decisions
4
 /speckit.plan สร ้ ำงแผนtechnical implementation — architecture, tech stack, API design
5
 /speckit.tasks แตก plan เป ็ นtasks ย่อย เรียงตำม dependency พร ้ อมบอกparallel [P] tasks
6
 /speckit.implement สั่ง Claude ท ำทุกtask ตำม plan + spec จน feature เสร็จสมบูรณ์
ติดตั้ง: specify init my-project --ai claude          โครงสร้ำง: .speckit/constitution.md, specifications/, plans/, tasks/

TDD Guard — Test-Driven Development
TDD Guard บังคับให้ Claude ท ำTDD อัตโนมัติ!
Block กำรเขียนโค้ดถ้ำยังไม่มี failing test
Block over-implementation เกินที่ test ต้องกำร
Red → Green → Refactor
R G Rf
🔴 RED เขียน Test ก่อน เขียน test ที่ describe behavior ที่ต้องกำร → รัน → ต้อง FAIL (เพรำะยังไม่มี code)
🟢 GREEN เขียน Code ให้ผ่าน เขียน minimal code เท่ำที่ test ต้องกำร ห้ำมเขียนเกิน → รัน → ต้อง PASS
🔵
REFACTOR ปรับปรุงโค้ด Refactor code ให้สะอำด ลด duplication, ปรับ structure → รัน test อีกที → ต้อง PASS
เครื่องมือ TDD ส าหรับ Claude Code:
TDD Guard npm i -g tdd-guard
Hook ที่ block code ถ้ำไม่มี test
TDG Plugin /plugin install tdg
Red-Green-Refactor + atomic commits
CLAUDE.md TDD เพิ่มใน CLAUDE.md
"Use TDD. Write test first, then code."

--- PAGE 11 ---
Best Practice Workflow — ครบทุกขั้นตอน
INIT 1
 Setup เขียน CLAUDE.md + .speckit/constitution.md ก ำหนดproject rules, tech stack, coding standards
SPEC 2
 Spec /speckit.specify → เขียน spec สิ่งที่ต้องกำร → /speckit.clarify → /speckit.plan → /speckit.tasks
GIT 3
 Branch git checkout -b feature/xxx — สร ้ ำงbranch ใหม่ แยกงำนจำก main เสมอ
TDD 4
 Test First 🔴 RED: เขียน test ก่อน → รัน → ต้อง FAIL (TDD Guard จะบังคับขั้นตอนนี้)
TDD 5
 Implement 🟢 GREEN: ให้ Claude เขียน minimal code ให้ test ผ่ำน → รัน → ต้อง PASS
TDD 6
 Refactor 🔵 REFACTOR: ปรับปรุง code quality, ลด duplication → test ยังต้อง PASS
QA 7
 Review ตรวจโค้ดทุกบรรทัด: logic, security, edge cases, performance → git add + commit
SHIP 8
 PR + Merge สร ้ ำงPR → Claude PR Review ตรวจอัตโนมัติ → แก้ feedback → merge เข้ำ main
INIT → SPEC → GIT → TDD (Red→Green→Refactor) → QA → SHIP   — ท าซ ้าทุก feature!

Ralph Loop — Autonomous AI Development
Official Anthropic Plugin — ให้ Claude ท ำงำนวนซ ้ำจนเสร็จ โดยไม่ต้องดูแล! เหมำะกับงำน refactor, migration, batch tasks ที่มี
success criteria ชัดเจน
วิธีท างาน:
1 คุณสั่ง /ralph-loop พร ้ อมprompt + completion criteria
2 Claude ท ำงำนตำมprompt → พยำยำม exit
3 Stop Hook ดักจับ exit → ส่ง prompt เดิมกลับเข้ำไปใหม่
4 วนซ ้ำจนกว่ำ Claude จะ output completion promise หรือถึง max iterations
ตัวอย่างค าสั่ง
/ralph-loop "Migrate tests
from Jest to Vitest.
Output <promise>DONE</promise>
when all pass"
--max-iterations 30
Best Practices
ใส่ --max-iterations เสมอ (ป ้ องกันinfinite loop)
เขียน success criteria ชัดเจนใน prompt
ใช้กับงำน greenfield / migration / batch tasks
หลีกเลี่ยงงำนที่ต้องใช้ judgment มำกๆ

Context Window, Limitations & Auto-Compact
Context = 200K tokens — ทุกไฟล์, command output,
MCP tools, CLAUDE.md กิน tokens ทั้งหมด!
Auto-Compact (เป ิ ดโดยdefault)
เมื่อ context ถึง ~75-95% → auto-compact จะท ำงำนอัตโนมัติ
สรุป conversation เก่ำ → เก็บเฉพำะข้อมูลส ำคัญ → ลบ messages เก่ำ
ข้อเสีย: อำจ "ลืม" รำยละเอียดส ำคัญที่ถูกสรุปทิ้ง!
ดีกว่ำ: สั่ง /compact เองตอนเปลี่ยนงำน ไม่ต้องรอ auto
วิธีจัดการ Context:
/clear ทุกครั้งที่เริ่มงำนใหม่
/context ดู token usage เสมอ (อย่ำเกิน 60-70%)
ใส่ข้อมูลถำวรไว้ใน CLAUDE.md ไม่ต้องพิมพ์ซ ้ำ
ใช้ Subagents แยกงำนใหญ่ออกเป ็ นcontext ย่อย
ใช้ -p ส ำหรับ one-shot (ไม่ต้องเป ิ ดsession)
ป ิ ดMCP ที่ไม่ใช้ — แต่ละตัวกิน 1-3% ของ context
⚠️ข้อจ ากัดที่ต้องรู ้ (Limitations)
1 Context Window 200K tokens
พอใช้งำนนำน context เต็ม → คุณภำพตก → ลืมค ำสั่ง
MCP tools + system prompt กิน ~10-25% ก่อนเริ่มงำน
2 Rate Limit (5-hr rolling window)
Pro: ~40-80 prompts/5hr, Max: 200-800 prompts/5hr
Weekly cap ด้วย — ใช้หมดต้องรอ reset
3 Hallucination (ค าตอบผิด)
Claude อำจแนะน ำcommand/option ที่ไม่มีจริง
ตรวจสอบ man page, verify ก่อน execute เสมอ!
4 ไม่มี Memory ข้าม Session
เริ่ม session ใหม่ = ลืมหมด (ต่ำงจำก Claude.ai)
เขียน CLAUDE.md เก็บ context ถำวร
5 Security Risks
Claude อ่ำน env vars, config files ได้
อย่ำใช้กับ production data โดยไม่ระวัง
6 Auto-Compact สูญเสียข้อมูล
เมื่อ compact อัตโนมัติ → สรุปอำจตกหล่น
Manual /compact ดีกว่ำ — ควบคุมได้เอง

CLI Patterns ที่นักพัฒนาต้องรู ้
Headless Mode (-p)
claude -p "มีกี่ไฟล์ในโปรเจกต์?"
รันแบบ one-shot ไม่ต้องเป ิ ดinteractive
Pipe Input
cat error.log | claude -p "วิเครำะห์ error นี้"
ส่ง output ของ command อื่นเข้ำ Claude
Chain Commands
claude "checkout new branch and lint"
สั่ง Claude ท ำหลำยtasks ต่อเนื่อง
File Reference
claude "อธิบำย src/api/auth.ts"
Tab-complete ชื่อไฟล์ในค ำสั่ง
